This documentation describes the Imaginea mViewer implementation.

It has the following sections:

  • Overview of the main pieces
  • Overview of the main Service classes
  • Overview of the main UI classes

Overview of the main pieces

mViewer exposes the db operations via stateless service clasess. The stateless services uses the provider MongoInstanceProvider to get information about the current mongo instance. The user interacts with a json over http layer implemented using request dispatchers.


Overview of the main Service classes

The service classes reside in the com.imaginea.mongodb.services package. They provide the actual functionalities of manipulating a collection or document or db. Classes are organized on the basis of the entity (like document, collection or database) they operate on.


Overview of the main UI classes

The UI backed is implemented using request dispatchers in the com.imaginea.mongodb.requestdispatchers package. Request Dispatchers are organized based on the utility they provide to the application like User login and logout, graphs, db, document and statistics.
